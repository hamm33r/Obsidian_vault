# 大小端模式
![](assets/数据的存储和排列/file-20260409191658803.png)
>地址是低到高排列的
>大端存储像人类正常看数字，低地址存的是最高有效字节，高地址存的是最低有效字节
>小段存储是地址从低到高，数字的权值也是从低到高

>小端存储是对于一个机器数内部的**字节之间**是逆序的
>比如一个4字节机器数0x 01 23 45 67 那么在小端存储中，假设地址从左到右递增依次是 0001 0002 0003 0004。
>那么在小端存储中实际存放的是 67 45 23 01，而不是 ~~76 54 32 10~~。
>小端存储的顺序只是一个数存储的字节之间的顺序，**不改变字节内部的顺序**
>并且小端存储只是针对一个单独的数据，不是针对数组的

![](assets/数据的存储和排列/file-20260414112829165.png)
>一个地址就是一个字节，一个short占两个字节，先找到-6，转为补码
>补码表示为0xFF FA，在小端存储中，字节之间是低权值的放前面，所以实际存的是
>0xFA FF(1个16进制位表示4bit，两个16进制位才是一字节)
# 边界对齐
![](assets/数据的存储和排列/file-20260409191658800.png)
>**存储字长**：存储单元中二进制代码的位数
>字长或机器字长指数据通路的宽度，等于CPU内部总线的宽度、运算器的位数、通用寄存器的宽度
>假设一个字为32位，那么**一个字=4个字节**。而计算器是按字节编址。那么字节1的地址就是0
>字节2的地址就是1，字节3的地址就是2，填充的地址就是3，第二行的字节1的地址就是4
>那么假设我要读2号字，我需要知道2号字的地址，第一行起始地址是0，第一行一整行是0号字
>第二行就是1号字，那么2号字就在第3行且起始地址为8（半字3的开头）。计算机在找2号字的过程中，会先找出它对应的编号，通过一个字等于4个字节，那么两个字就是8个字节，算出8的过程就是通过对2$(10)_2$（因为是2号字）进行逻辑左移两位，相当于乘4得到8。那么就找到了2号字的起始地址

>存储对齐：跟结构体的内存对齐和访存次数有关。因为每次访存都只能读/写1个字。假设第一行前3个字节都被char填充，接下来要往这个结构体里再放一个short，如果不对齐，那么就得把short分两半，一半存在第一行最后一格，另一半存在第二行第一格，这样访存的时候就得访问两次，所以直接把short放在第二行，这样访存一次就行了
>内存对齐是一种**以空间换时间的策略**

>按边界对齐方式存储：**数据的存储地址是其自身大小（自身字节长度）的整数倍**
>如果是char类型的数据，就可以存在 0000H,0001H,0002H
>但如果是int类型的数据,就只能存在 0000H,0004H,0008H

## 结构体的内存对齐规则
1. 结构体的第一个成员对齐到和结构体变量起始位置偏移量为0的地址处
2. 其他成员变量要对齐到**对齐数**的整数倍的地址处
	- 对齐数=编译器默认的一个对齐数与该成员变量大小的较小值
	- ~~VS中默认的对齐数为8~~
	- 对齐数就是成员自身的大小
3. 结构体总大小为**最大对齐数**(结构体中每个成员变量都有一个对齐数，所有对齐数中最大的)的整数倍
4. 如果嵌套了结构体的情况，嵌套的结构体成员对齐到自己的成员中最大对齐数的整数倍处，结构体的整体大小就是所有最大对齐数（含嵌套结构体中成员的对齐数）的整数倍
5. **数组成员变量**在结构体里，只有数组的第一个元素按照自身的类型进行对齐，剩下的直接按照数组长度平铺，比如`int arr[10]`那么只有`arr[0]`进行int变量的对齐，然后剩下的元素直接连续存放。

# 存储字长，存储字数，机器字长
在计算机组成原理中，**存储字长**和**存储字数**是描述主存储器特性的两个核心概念，它们与**MDR**（存储器数据寄存器）和**MAR**（存储器地址寄存器）有着密切的对应关系，共同决定了主存储器的容量和访问特性。

### 一、基本概念解析

#### 1. 存储字长

- **定义**：存储字长是指**一个存储单元中存储的二进制代码的位数**，即单次访问主存可获取的数据量。
- **常见规格**：通常为8位、16位、32位或64位，必须是字节（8位）的整数倍。
- **意义**：它决定了主存一次读写操作能传输的数据量，直接影响内存带宽和访问效率。例如，32位存储字长表示每次内存访问可传输32位（4字节）数据。
- 决定了一个字包含多少个字节

#### 2. 存储字数

- **定义**：存储字数是指**主存储器中存储单元的总数**，即主存能容纳多少个存储字。
- **计算方式**：存储字数 = 2MAR位数，例如MAR为32位时，存储字数 = 232 = 4G个存储单元。
- **意义**：它反映了主存的寻址能力，决定了系统能支持的最大内存容量。

#### 3. 主存总容量

- **计算公式**：主存总容量 = **存储字数 × 存储字长**。
- **示例**：若MAR为32位（存储字数=4G），MDR为8位（存储字长=8位），则总容量 = 4G × 8位 = 32Gb = 4GB。
- **单位转换**：注意区分位（bit）与字节（Byte），1Byte = 8bit，且K/M/G在存储容量中表示1024倍关系（210/220/230）。

### 二、与MDR和MAR的对应关系

#### 1. MDR与存储字长

- **核心关系**：**MDR位数 = 存储字长**。
- **功能解释**：MDR（存储器数据寄存器）用于暂存从存储体中读出的数据或准备写入存储体的数据，其位数必须与存储单元的位数完全匹配。
- **工作过程**：
    - 读操作时：存储体 → MDR（数据被读入MDR）
    - 写操作时：MDR → 存储体（数据从MDR写入存储体）
- **示例**：若存储字长为32位，则MDR必须是32位寄存器，确保能完整容纳一个存储单元的数据。

#### 2. MAR与存储字数

- **核心关系**：**MAR位数 = log₂(存储字数)**，即存储字数 = 2MAR位数。
- **功能解释**：MAR（存储器地址寄存器）用于存放访问的存储单元地址，其位数决定了可寻址的存储单元数量。
- **工作过程**：
    - CPU将地址送入MAR → MAR通过地址总线访问存储体 → 选中对应存储单元
- **示例**：若MAR为28位，则可寻址228 = 256M个存储单元（如主存容量为256M×32位）。

### 三、协同工作示例

以**取数操作**为例，说明四者如何协同工作：

1. CPU将指令地址送入MAR（地址寄存器）
2. MAR通过地址总线访问存储体，选中对应存储单元
3. 存储单元中的数据（长度=存储字长）被读入MDR
4. MDR将数据送入CPU进行处理（如存入ACC寄存器）

以**存数操作**为例：

1. CPU将数据地址送入MAR
2. CPU将数据送入MDR
3. MAR通过地址总线选中存储单元
4. MDR中的数据被写入存储单元（长度=存储字长）

### 四、重要补充说明

#### 1. 存储字长与机器字长的关系

- **机器字长**：CPU一次能处理的数据位数（如32位/64位）
>	说一台计算机是32位/64位，指的就是机器字长
- **关系**：存储字长**通常等于**机器字长，但**可以不同**。
    - 若存储字长 < 机器字长（如32位CPU配16位内存）：需多次访问内存，影响性能
    - 若存储字长 > 机器字长：浪费总线带宽，但可提高指令密度

#### 2. 存储字长与指令字长的关系

- **指令字长**：单条指令的二进制位数
- **关系**：指令字长**必须是存储字长的整数倍**。
    - 若指令字长 = 存储字长：一次访问可取完整指令
    - 若指令字长 = 2×存储字长：需两次访问才能取完整指令

#### 3. 实际应用中的灵活性

- 现代计算机主存通常**按字节编址**，但支持**按字节、半字或字访问**。
- 通过**地址移位**实现不同粒度访问：
    - 字节地址 → 右移1位 = 半字地址
    - 字节地址 → 右移2位 = 字地址
- **大端/小端模式**：影响多字节数据在存储单元中的排列方式，但不改变存储字长本质

### 五、典型实例分析

**例1**：某计算机MAR为32位，MDR为64位

- 存储字数 = 232 = 4G个单元
- 存储字长 = 64位（8字节）
- 总容量 = 4G × 64位 = 32GB
- 特点：64位存储字长匹配64位CPU，一次访问可传输8字节数据，效率高

**例2**：某嵌入式系统MAR为16位，MDR为8位

- 存储字数 = 216 = 64K个单元
- 存储字长 = 8位（1字节）
- 总容量 = 64K × 8位 = 64KB
- 特点：8位存储字长适合低功耗场景，但需多次访问处理32位数据
